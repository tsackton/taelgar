After reviewing a session recap, you can regenerate the session material with:
```
 python _scripts/build_session_note_components.py --overwrite -c <campaign> -n <sessionNumber>
```

Run  `python _scripts/build_session_note_components.py --help` for a list of campaigns.

Missing campaigns can be added to: `_scripts/session_note_campaigns.json`:

- `sessionRoot` is the base name for generated components
- `aliases` is the short names for identifying the campaign
- `campaignRoot` is the campaign directory in Obsidian relative to the vault root
- `notePattern` is the template for generated session note names; recognizes {session} or {session_padded} as number replacements
- `defaultTemplate` is the template file in `_templates/session-notes` to use

The session note itself can be generated by running the render-session-note templater template.

The Python script reads from the session-recap
The templater script reads from the `_generated` components
