[2026-04-08 08:41 AM] rsulfuratus: okay - so the whole system is kind of in place now. still ironing out some bugs but the basic framework:
    
    (1) agentic tools clean transcript (or raw notes/narrative if desired, e.g. can use Cleenseau blog posts as input), split into session beats, extract facts from the session beats (NPCs, locations, items, dates, though dates requires the start/end date of the session be input), and generate a session recap with facts and narratives
    
    (2) human edits the session recap to fix errors, add links, clean things up
    
    (3) the python code build_session_note_components.py in _scripts generates composable segments from the session recap, and creates a stub note for the session. this is controlled by command line options and the session_note_campaigns.json
    
    (4) templater uses render session note to generate note based on the template
    
    likely still a few bugs to work out and needs actual documentation but the basic idea is that everything after the human review of session recap is deterministic / non-LLM, so once the session recap is clean note generation is more or less automatic.
